libARMavlink Documentation

Introduction

The ARMavlink library provides an API to read and write mavlink files

The library uses ARMAVLINK_FileGenerator_t object as its main entry point to write mavlink files The library uses ARMAVLINK_FileParser_t object as its main entry point to read mavlink files

ARMavlink writing file usage

The typical use flow of the ARMavlink library to write a mavlink file is the following:

1> Create the File generator

The ARMAVLINK_FileGenerator_t can now be properly created, using the ARMAVLINK_FileGenerator_New function.

2> Use the File generator

At this point the ARMAVLINK_FileGenerator_t is ready for use. You can add waypoints to the list, replace ones, also insert or delete waypoints. Then you can call ARMAVLINK_FileGenerator_CreateMavlinkFile to create the mavlink file.

2> Close the File generator

The ARMAVLINK_FileGenerator_t object can the be destroyed using ARMAVLINK_FileGenerator_Delete function.

This call is mandatory, as it reclaims all allocated memory.

ARMavlink reading file usage

The typical use flow of the ARMavlink library to read a mavlink file is the following:

1> Create the File parser

The ARMAVLINK_FileParser_t can now be properly created, using the ARMAVLINK_FileParser_New function.

2> Use the File parser

At this point the ARMAVLINK_FileParser_t is ready for use. All you have to do is to call ARMAVLINK_FileParser_Parse This gives you a list of waypoints

2> Close the File parser

The ARMAVLINK_FileParser_t object can the be destroyed using ARMAVLINK_FileParser_Delete function.

This call is mandatory, as it reclaims all allocated memory.
